# Gitmoji + 日本語コミット

ステージされた変更を、先頭に Gitmoji（Unicode 絵文字）+ 日本語の簡潔な description でコミットする。GPG 署名を付け、push は明示的に依頼されたときだけ行う。

## フォーマット

```
<gitmoji> <日本語 description>
```

- **コミットメッセージは必ず 1 行で完結させる。本文・footer は使わない。**
- **必ず Unicode 絵文字（例: `✨`）を使う。`:sparkles:` のような shortcode 形式は使わない。** GitHub などは shortcode を描画するが、GitLab や一部のホスティング・ターミナル・エディタでは描画されず文字列のままになるため。
- `絵文字 + 半角スペース + 日本語の主題` の形にする。
- description は体言止めまたは簡潔な動詞句で、句点（。）は付けない。
- 50 文字程度を目安に、長くても 72 文字を超えない。
- スコープを示したいときは `<gitmoji> [scope] description` の形にしてもよい。

## 作業手順

1. `git status` でステージ済み・未ステージの変更を把握する。
2. `git diff --cached`（必要なら `git diff`）で変更内容を確認する。
3. 直近のコミット履歴 `git log --oneline -10` でリポジトリの絵文字運用を確認する。
4. 変更の主目的に最も合う絵文字を1つだけ選ぶ。表で迷う場合や該当が見当たらない場合は https://gitmoji.dev/ で公式の絵文字一覧と意味を確認する。
5. 日本語で何をしたかを短く一文で表現する（敬体は使わない）。1 行で説明しきれない場合は、コミットを分割するか description を磨き直すことを優先し、複数行にはしない。
6. GPG 署名設定を確認してからコミットする。

## メッセージ例

### 良い例

```
✨ ログイン機能を追加
🐛 トークン期限切れ時の NPE を修正
📝 README にインストール手順を追記
♻️ ユーザーサービスをリポジトリ層に分離
🔧 ESLint 設定をフラットコンフィグへ移行
⬆️ React を 19.0.0 に更新
🔥 廃止された v1 API クライアントを削除
💥 設定ファイルのキー名を刷新
```

### 避けるべき例

- 絵文字なしや絵文字 2 個以上の併用
- `:sparkles: 新機能を追加`（shortcode 形式。描画されない環境がある）
- 改行を含む複数行のコミットメッセージ（本文・footer を付ける運用）
- `✨ 新機能を追加しました。`（敬体・句点）
- `✨ feat: ログイン追加`（Conventional Commits 形式の type と混在させない）
- `✨ いろいろ修正`（具体性なし）

## コミット実行

GPG 署名付きでコミットする。常に 1 行のメッセージのみを使う。

```sh
git commit -m "<gitmoji> <日本語 description>"
```

## 安全上のルール

- push は明示的に依頼されたときだけ行う。
- `--no-verify` や `--no-gpg-sign` は使わない。
- amend は依頼されたときだけ行う。新規コミットを基本とする。
- 秘密情報やトークンをコミットメッセージに含めない。
- ステージされていない変更を勝手に `git add` で取り込まない。範囲はユーザーに確認する。
- Conventional Commits 形式とは混在させない。両方使いたい場合はリポジトリのルールに合わせる。
